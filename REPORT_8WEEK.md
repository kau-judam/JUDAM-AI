## 1. 주요 진행 내용

### 1.1 데이터 수집 및 전처리
- **데이터 소스**: 한국농수산식품유통공사 전통주 정보 API
- **수집 데이터**: 207개 막걸리 제품 정보
- **전처리 완료**: 중복 제거, 결측치 처리, 텍스트 정제

### 1.2 맛 벡터 추출 시스템 고도화
- **기존 문제**: 45.4% 중복 → 추천 정확도 저하
- **고도화 방안**:
  1. 벡터 세분화 (5단계 키워드 분류)
  2. 향 노트 차원 추가 (4개 카테고리, 15개 세부 항목)
  3. NaN 데이터 처리 (null 변환)
- **개선 결과**: 중복 비율 45.4% → 8.7% (벡터+노트 기준)

### 1.3 추천 시스템 구현
- **코사인 유사도 기반 추천**: 8차원 맛 벡터
- **취향 진화 트래킹**: 사용자 평가 기반 맛 벡터 자동 업데이트
- **역추천**: 음식/안주 기반 전통주 추천
- **다중 소스 앙상블**: 맛(70%) + 원재료(20%) + 지역(10%)

### 1.4 FastAPI 서버 구축
- **엔드포인트**:
  - `POST /api/recommend`: 맛 벡터 기반 추천
  - `POST /api/taste/update`: 사용자 취향 업데이트
  - `GET /api/taste/history/{user_id}`: 취향 히스토리 조회
  - `POST /api/food/recommend`: 음식 기반 추천
  - `POST /api/survey/convert`: 술BTI 설문 → 맛 벡터 변환

### 1.5 프로젝트 구조 정리
- **폴더 구조**: core(핵심 로직), crawler(데이터 수집), utils(유틸리티), tests(테스트)
- **불필요한 파일 삭제**: 중복/테스트/시뮬레이터 파일 11개 정리
