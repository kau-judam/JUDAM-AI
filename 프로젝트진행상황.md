## 2. 문제점 및 불필요한 파일

### 2.1 불필요한 파일 (삭제 필요)

| 파일명 | 이유 |
|--------|------|
| `app/filter.py` | `app/law_client.py`와 중복 (law_client가 더 완성됨) |
| `app/utils/data_preprocessor.py` | `app/auto_pipeline.py`가 CSV 파싱 담당 |
| `app/crawler/multi_source_crawler.py` | SKILL.md에 없음, AI 파트 범위 벗어남 |
| `tests/` 디렉토리 전체 | 요청 없음 |
| `pytest.ini` | 테스트 없음 |
| `Dockerfile` | 배포 관련, AI 파트 범위 벗어남 |

### 2.2 중복 모듈 정리

- `app/filter.py` vs `app/law_client.py`
  - `law_client.py` 유지 (SKILL.md 요구사항에 맞게 구현됨)
  - `filter.py` 삭제

- `app/utils/data_preprocessor.py` vs `app/auto_pipeline.py`
  - `auto_pipeline.py` 유지 (CSV 파싱 + Gemini 라벨링 + 맛 벡터 생성)
  - `data_preprocessor.py` 삭제

## 3. 수동으로 해야 할 작업

### 3.1 필수 작업

1. **`.env` 파일 생성**
   ```bash
   cp .env.example .env
   ```
   - `GEMINI_API_KEY`: Gemini API 키 입력
   - `LAW_API_KEY`: 국가법령정보센터 API 키 입력
   - `DATABASE_URL`: PostgreSQL DB 연결 정보 (선택)
   - `REDIS_URL`: Redis 연결 정보 (선택)

2. **의존성 설치**
   ```bash
   pip install -r requirements.txt
   ```

3. **앙커 데이터 입력** (`data/anchors.json`)
   - 실제 시음 기반으로 10~20개 앙커 데이터 입력
   - 형식:
     ```json
     {
       "anchors": [
         {
           "name": "복순도가 복분자주",
           "sweetness": 8.0,
           "body": 5.0,
           "carbonation": 2.0,
           "flavor": 9.0,
           "alcohol": 7.0,
           "acidity": 7.0,
           "aroma_intensity": 8.0,
           "finish": 6.0,
           "note": "실제 시음 기반"
         }
       ]
     }
     ```

### 3.2 권장 작업

4. **데이터 파이프라인 실행**
   ```bash
   python app/auto_pipeline.py
   ```

5. **서버 실행**
   ```bash
   uvicorn app.main:app --reload --port 8000
   ```

6. **API 테스트**
   - http://localhost:8000/docs 접속
   - 각 엔드포인트 테스트

### 3.3 선택 작업

7. **DB 생성** (PostgreSQL 사용 시)
   ```sql
   CREATE DATABASE juddam;
   ```

## 4. API 엔드포인트 목록

| 엔드포인트 | 메서드 | 설명 |
|------------|--------|------|
| `/` | GET | 루트 |
