## 전체 기능 구성

### 1. 추천 시스템
- 맛 벡터 기반 코사인 유사도 추천
- 다중 소스 앙상블 (맛 + 원재료 + 지역)
- 컨텍스트 기반 추천 (계절/상황/기분)
- 취향 진화 트래킹 (사용자 평가 기반)
- 음식 기반 역추천 (안주 매칭)

- 5개 축 설문 (공개용)
- 8개 축 내부 사용 (추가 정보)
- 설문 응답 → 맛 벡터 변환 로직

### 3. 인사이트 대시보드
- 현재 집계 (Pandas)
- 미래 예측 (Holt-Winters)
- 레시피 군집화 (K-Means)

### 4. 법률 필터링
- 국가법령정보센터 API
- RAG 검색
- Gemini 판단

## 술BTI 구성 (제안)

### 공개용 5개 축
1. **단맛 (sweetness)**: 0~10
2. **바디감 (body)**: 0~10
3. **탄산 (carbonation)**: 0~10
4. **풍미 (flavor)**: 0~10
5. **도수 (alcohol)**: 0~10

### 내부용 8개 축 (추가)
6. **산미 (acidity)**: 0~10
7. **향기 강도 (aroma_intensity)**: 0~10
8. **여운 (finish)**: 0~10

### 설문 문항 (제안)
1. "달콤한 술을 선호하시나요?" (단맛)
2. "묵직하고 걸쭉한 술을 선호하시나요?" (바디감)
3. "탄산감이 있는 술을 선호하시나요?" (탄산)
4. "풍미가 풍부한 술을 선호하시나요?" (풍미)
5. "도수가 높은 술을 선호하시나요?" (도수)
