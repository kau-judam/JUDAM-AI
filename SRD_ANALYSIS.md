## 3. SRD(Software Requirements Document) 필요 여부

### 3.1 SRD가 필요한 이유

3. **테스트 기준**: 기능 테스트 및 인수 테스트 기준 확립
4. **유지보수**: 추후 기능 추가 및 수정 시 참고 자료

## 4. 개발 계획서/설계 파트 문제점 분석

### 4.1 현재 문제점

| 문제점 | 설명 | 해결 방안 |

#### 4.2.2 AI 서비스 API 명세서

| 엔드포인트 | 메서드 | 설명 | 요청 | 응답 |
|------------|--------|------|------|------|
| `/api/survey/convert` | POST | 술BTI 설문 → 맛 벡터 변환 | 설문 응답 | 맛 벡터 |
| `/api/recommend` | POST | 맛 벡터 기반 추천 | 맛 벡터, top_k | 추천 결과 리스트 |
| `/api/taste/update` | POST | 사용자 취향 업데이트 | user_id, drink_id, rating, tags | 업데이트 결과 |
| `/api/taste/history/{user_id}` | GET | 취향 히스토리 조회 | user_id | 히스토리, 진화된 맛 벡터 |
| `/api/food/recommend` | POST | 음식 기반 추천 | food, top_k | 추천 결과 리스트 |

#### 4.2.3 Node.js ↔ FastAPI 통신 방식

```javascript
// Node.js (백엔드)
const axios = require('axios');

// 술BTI 설문 → 맛 벡터 변환
async function convertSurveyToVector(surveyResponse) {
  const response = await axios.post('http://localhost:8000/api/survey/convert', surveyResponse);
  return response.data.taste_vector;
}

// 맛 벡터 기반 추천
async function getRecommendations(tasteVector, topK = 10) {
  const response = await axios.post('http://localhost:8000/api/recommend', {
    user_vector: tasteVector,
    top_k: topK
  });
  return response.data;
}
```
