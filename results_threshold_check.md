# BTI 룰 임계값(5.0) 실효성 점검 — 실사용자 76명 실측

- **데이터**: `술bti실효성검증.xlsx` `responses` 76행의 8축 실측값(조작·가정 없음, 실측만).
- **점검 대상**: 룰 기반 BTI 4글자 코드의 축별 임계값(현재 4축 모두 **5.0**).
  - sweetness ≥5 → `S` / else `D`
  - body ≥5 → `H` / else `L`
  - carbonation ≥5 → `F` / else `M`
  - flavor ≥5 → `U` / else `C`
- **검증**: 위 룰로 8축에서 재계산한 4글자 코드가 시트의 `bti4`와 **76/76 완전 일치** → 룰 재현 정확.
- **전제 고정**: BTI 코드 체계·16타입·캐릭터는 **변경하지 않음**. 본 문서는 분석·제안만이며 **코드 변경 0**.
- ⚠️ aroma_intensity(평균 6.72)는 높지만 **코드 축이 아님** → 임계 판정에서 제외(참고용).

## 4. 결론 — 임계 5.0 적절성 판정

### 판정: **현 표본 기준 5.0은 다소 낮음(조정 여지 있음). 단, 지금 바꾸는 것은 권장하지 않음.**

**왜 5.0이 이 데이터엔 부적절한가 (실측 근거)**
- 네 룰축 **중앙값이 모두 5.0 초과**(5.68~6.57) → 임계선이 분포 중심보다 아래.
- 그 결과 **모든 축이 高글자로 쏠림**, 특히 **sweetness 82%가 S**로 변별력 상실.
- 16타입이 SHFU(25%) 등 소수 셀에 집중되고 **1명짜리 타입 6개** 발생 → 유형화의 정보량 저하(엔트로피 0.833).
- 이는 앞선 경계 분석(룰축의 29~43%가 임계 5 근처)과 일관 — 5.0이 표본을 잘 못 가른다는 정황.

**그럼에도 지금 조정하지 말아야 하는 이유**
1. **표본 편향·소표본**: 76명은 단맛 선호 쪽으로 치우친 지인 중심 비무작위 표본. 중앙값(6.57 등)은
   이 표본에 과적합될 수 있어, 실제 사용자 모집단을 대표한다는 보장이 없음.
2. **대규모 재분류 충격**: 임계를 중앙값으로 바꾸면 **기존 76명 중 41명(54%)의 BTI 4글자 코드가 바뀜.**
   (축별 글자 뒤집힘: sweetness 23, flavor 15, body 11, carbonation 7명)
   → 이미 결과·캐릭터를 받은 사용자 절반의 유형/캐릭터가 변경되는 UX 혼란.
3. **공집합 발생**: 중앙값 기준에서 DLMU 타입이 0명 → 단순 중앙값 일괄 적용도 완벽한 해법은 아님.
4. BTI 코드 체계·캐릭터는 고정이므로, 임계만 바꿔도 사용자가 보는 "내 유형"이 흔들림.

### 제안 (적용하지 않음 — 효과/영향만 명시)
| 방향 | 이렇게 하면 | 효과 | 기존 76명 영향 |
|------|------------|------|----------------|
| A. 유지(권장) | 5.0 고정 유지 | 결과 안정, 기존 사용자 유형 불변 | 0명 변경 |
| B. 축별 중앙값 임계 | sweetness 6.57·body 5.85·carb 5.68·flavor 5.79 | 분포 균등화(std 5.07→2.54, 1인타입 6→0) | **41명(54%) 코드 변경**, DLMU 공집합 |
| C. 분위수/표준화 정규화 | 각 축을 관측분포로 정규화 후 50백분위 분기 | 표본 변화에 자동 적응 | 데이터 갱신마다 변동(고정 임계 아님) |
| D. 데이터 기반 학습 | KNN 등으로 경계 학습(FUTURE_WORK 2-1) | 경계 케이스 정밀화 | 모델 갱신 시 변동 |

**권고**: 발표/현 단계는 **5.0 유지(A)**. 표본이 더 크고 대표성을 갖출 때까지 임계 조정 보류.
중기적으로는 고정 5.0 대신 **C(분위수 정규화) 또는 D(학습 기반)** 로 전환 검토 — 단맛 축의 쏠림(82% S)이
가장 먼저 손봐야 할 지점. 어떤 경우든 **임계 변경 = 기존 사용자 절반의 유형 변경**임을 사전 고지 필요.
